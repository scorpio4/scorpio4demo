Assets

Assets are text-based files that play a key role in Scorpio4. Assets are roughly divided into two categories - scripts & queries and everything else.

Assets are distinguished by their mime type, a URI that describes how the contents should be understood.


Scripts

Scripts are mini-programs, they are triggered at useful times when your application runs.

How and when each script is run is described by rules that are stored in your local repository.

Scorpio4 can run your scripts whenever you need them. Linking scripts into execution pipelines allows small scripts to play a big role. They make your software easier to build and maintain.

Scripts can be run synchronously or asynchronously, depending upon the execution engine that is interpreting them.


Queries

Queries are instructions to create, read, update or delete data in some data source. Scorpio4 supports SQL and SPARQL queries. Other query engines are planned.
