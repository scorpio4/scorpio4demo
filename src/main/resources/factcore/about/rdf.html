Scorpio4 RDF

We want to avoid ambiguity when sharing and consuming data. One solution is to use a URI as an identifier.

If two or more parties use the same identifier, they readily agree they are talking about the same thing.

RDF is a data modelling language that uses URIs to describe everything.

It's schema language, RDFS is used to construct vocabularies where all the terms are defined by URIs.

The definition of each URI can usually be found by issuing an HTTP request for the URI.

This decentralised, self-referencing, self-describing, ad-hoc data model is the chosen data model for the semantic web and linked data.

It was designed to allow computers to produce and consume data without significant human intervention.

RDF uses URIs so that a computer can potentially discover the meaning of any identifier, making RDF an ideal choice for interoperability and API-as-a-service based solutions.

We hope that Scorpio4 plays a small role in promoting the myriad of use cases that Linked Data enables and simplies.

Scorpio4 Facts vs RDF

To keep things simple, we adopt the terms 'hard facts' and 'literal facts'.

In all cases, facts are simply RDF triples that omit some of nuances for sake of clarity when talking to non-geeks.

Hard Facts

A "Hard Fact" is a simple statement comprised of 3 distinct URIs - a subject and it's relationship (predicate) to an object.

 :Lee :drinks :Black_Coffee.

Literal Facts

A "Literal Fact" is a simple statement comprised of 2 distinct URIs - a subject and it's relationship (predicate) to a literal.

 :Lee :first_name  "Lee"^^xsd:string.

Where we have declared that resource :Lee has a XSD-string encoded literal as his :first_name

Inferencing

If you think about relationships, they tell you a lot about the two parties. In other words, we can make inferences about the subjects and objects by referencing the predicate.

We like to think of it as data-driven common sense.

Linked Data

Each atomic element of the statement is actually a URI - a globally unique identifier just like a web page. In fact some web pages already support embedded RDF.

http://example.com/whatis/Black_Coffee

Since each URI is globally addressable, it's possible to request more RDF that further expands the definition of :BlackCofee - perhaps that is suppliers, recipes or health related

From the RDF that the Black_Coffee site provides about it's beverage, we find more statements. We can continue like this until all the world's relevant linked data is consumed.

It's the links that matter. Those relationships provide meaning - hence the semantic web. Since all data can theoretically be called upon to explain itself, it makes for a very powerful knowledge management framework.

To Schema or Not Schema

A schema in SQL is a set of constraints that are imposed on the structure of the data. With RDF, a schema is a logical set of rules that infer further facts.

Furthermore, the data and meta-data are indistinguishable - all the tools and techniques that work with data also apply to meta-data.

RDF is flexible and simple enough that it can describe queries, data models, user interfaces, validation rules, business processes, integration patterns and more.

Scorpio4 take those possibilities and provides the infrastructure and tooling needed to design, test, deliver, deploy and maintain sophisticated systems that rely upon smart data rather than smart code.
